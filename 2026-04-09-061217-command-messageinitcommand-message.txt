⏺ Design Section 2: Data Flow

  1. CSV Input — A collapsible <textarea> at the top. User pastes CSV text. On
  change, the app parses it with a lightweight parser (no dependency — the
  format is simple enough for vanilla JS).
  2. Column Mapping — After a valid parse, two <select> dropdowns appear:
    - Date column — defaults to transaction_date if found, otherwise first
  column
    - Stack by — user picks any column (e.g., account, category)
  3. Filtering — Checkboxes read from the data's unique statement_type / account
   values:
    - Show all CC — toggle individual credit card accounts vs. combined
    - Replace CU pay with CC details — when a CU row is a credit card payment,
  swap it out for the matching CC transactions
    - Pick CC to stack — multi-select of which CC accounts to include
  4. Aggregation — Rows grouped by month (from the chosen date column), summed
  by the chosen stack column. Positive amount → income bucket; negative →
  expense bucket.
  5. Chart render — Chart.js receives one dataset per unique stack value,
  rendered as a stacked bar chart binned by month. Scrollable horizontally if
  many months.
  6. Summary panel — Totals for the selected month range: income (monthly avg +
  annual), expenses, net.

  Does this data flow match your mental model?

⏺ Design Section 3: Chart Behavior & Interactions

  Chart type: Stacked bar chart (Chart.js), one bar per month, one dataset per
  stack-by value. X-axis = months, Y-axis = dollar amounts.

  Scroll: If months exceed the viewport width, the chart container scrolls
  horizontally. The chart canvas is rendered at full width (all months) and the
  container clips it.

  Hover interactions:
  - Chart.js tooltip plugin shows a floating detail box on hover: amount per
  series (with color swatch), plus total. This maps to the drawing's "amount
  black / blue / red / total."
  - Non-hovered series are dimmed via Chart.js's onHover callback updating
  dataset backgroundColor opacity — other stacks drop to ~20% opacity while the
  hovered one stays full.

  Color key: Rendered below the chart as a legend — one colored swatch + label
  per stack series, each with a checkbox to toggle that series on/off. This
  replaces Chart.js's built-in legend.

  Does this cover the chart behavior correctly?
